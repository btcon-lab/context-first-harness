# docs/context — 컨텍스트 자산 (Single Source of Truth)

이 디렉토리는 {PROJECT} 하네스의 **컨텍스트 엔지니어링 자산**이다. 하네스(`.claude/agents`, `.claude/skills`)가 "누가·어떻게"라면, 여기는 에이전트들이 참조하는 **"무엇(정본 사실)"**을 담는다.

## 정본(SoT) 규칙
1. **같은 사실은 한 곳에만.** 각 계약은 아래 파일 하나가 유일한 정본이다. 에이전트는 이 파일을 **읽어** 참조하고 산출물에 복제하지 않는다(복제는 drift·경계면 버그의 근원).
2. **변경은 정본에서.** 계약을 바꾸려면 이 파일을 고치고 소비자에게 통지한다. 산출물에서 계약을 임의로 바꾸지 않는다.
3. **정본 ↔ 검증.** QA는 각 산출물이 정본과 일치하는지 교차 검증한다. 불일치는 "정본 위반".
4. **추적성.** 모든 계약 항목은 `fr-index.md`로 앵커링한다.

## Context(seam) vs Design(elaboration) — 오버스펙 방지 경계
`docs/context`는 **계약(seam)만** 담는다. 계약 = 2개 이상 에이전트가 공유하는 인터페이스 사실. 상세 설계(how)는 `docs/design`에 두고 **여기를 링크**한다. 재기술(복제) 금지.

| context(seam)에 두는 것 | design으로 보내는 것 (context를 링크) |
|---|---|
| 엔티티·키필드·제약 | 타입·인덱스·성능 튜닝 |
| 상태명·허용 전이·사유코드 | 전이 가드·사이드이펙트·구현 |
| 인터페이스 목록·응답 형식·규약 | body 스키마·내부 구현 |
| 역할×리소스×액션 | 강제 구현(미들웨어/정책 코드) |
| 지표명→산식 | 수집·집계 구현 |

**판단 기준:** "경계면(2+ 소비자)이 공유하는 사실인가?" → context. "한 소유자의 구현 방법인가?" → design.

## 변경 프로토콜 (CE 우선 · Contract-First) — 최우선 규칙
설계든 개발이든 계약에 영향을 주는 수정이 발생하면 **정본을 가장 먼저 고친다.** 순서:
1. **정본 갱신** → 2. **통지**(소유자→소비자) → 3. **반영**(design·코드) → 4. **검증**(QA 교차) → 5. **기록**(아래 변경 이력).

**Drift 판정:** 코드/산출물이 정본과 어긋나면 **정본이 진실**. 정본을 우회한 변경은 반려. **예외:** 순수 구현 세부(계약·상태·권한·지표 미영향)는 대상 아님.

## 두 개의 루프 — 구현을 고치는가, 계약을 고치는가

| | 즉시 교정 (Runtime) | 계약 개선 (Learning) |
|---|---|---|
| 고치는 대상 | **구현** | **정본** |
| 발동 | 판정 V(구현 위반) | 판정 G·S·D |
| 승인 | 불필요 — 계약이 이미 정답 | **사람 승인 필수** |

## 정본 변경 승인 게이트

계약은 아키텍처 결정이다. 에이전트가 검증 결과만 보고 정본을 고치면 **"구현에 맞춰 계약을 낮추는" 방향으로 흘러가고**, 정본은 코드의 사후 요약으로 전락한다.

```
G·S·D 판정 → QA가 [제안] → 소유자가 초안 → ★사람 승인★ → 반영·이력 → 통지 → 재검증
```

**제안에 담을 것:** 대상 정본·Rule ID / 현재 → 제안 / 판정 근거 / 영향 받는 소비자 목록.

**승인 없이 가능:** `[TODO]`를 처음 채우는 경우, 오타·서식·링크 교정.
**반드시 승인:** 규칙의 추가·완화·삭제. 특히 **완화(MUST → SHOULD 또는 삭제)** 는 가장 엄격히 본다 — 규칙이 틀린 것인지 팀이 못 지키는 것인지는 사람이 판단한다.

## 상태 표기 규약
`[정본]` 확정 사실 · `[TODO]` 팀이 채울 자리 · `[OPEN]` 착수 전 결정 필요(임의 결정 금지) · `[ASSUMPTION]` 근거 없는 가정(검증 대상).

## 규칙 등급
`[MUST]` 정책(`{SEAM}-P###`, override 불가, 위반 시 FAIL) · `[SHOULD]` 지침(`{SEAM}-G###`, 이유 명시 시 override 가능, 이탈 시 WARN) · 등급 없음(구현 자유, 판정 대상 아님). Rule ID는 삭제 후에도 재사용하지 않는다.
