# P-A: 입력 큐레이션 (상세)

무거운 원문을 매번 컨텍스트에 넣는 것은 토큰 낭비다. 한 번만 파싱→섹션 분리하고, 각 에이전트가 자기 섹션만 로드하게 한다.

## 절차
1. 원문 확인(.docx / .md / .txt). `.docx`는 바이너리이므로 스크립트로 텍스트 추출.
2. 스크립트 실행:
   ```
   python3 skills/context-first-harness/scripts/split-source.py <원문> [출력경로]
   # 기본 출력: docs/context/plan-extracts/
   ```
3. 결과 확인 — 섹션이 과분할/미분할이면 원문 헤딩 스타일을 보고 스크립트의 분리 규칙(번호 헤딩 단조 증가, md 헤딩)을 조정.
4. 각 `NN_*.md` 상단에 "정본 아님·읽기용" 배너가 있는지 확인.

## 규칙
- **읽기용·고정.** plan-extracts는 원문의 사본이므로 수정하지 않는다. 원문이 개정되지 않으면 재생성 불필요.
- 배선(P-D) 시 각 에이전트 스킬에 "계획서 원문은 `plan-extracts/NN_*.md`만" 이라고 자기 섹션을 지정한다.
- 원문과 정본을 대조하면 "원안 대비 무엇이 바뀌었나"를 추적할 수 있다(plan-extracts=원본, docs/context=현재 결정).

## 입력이 문서가 아니라 한 문장일 때
P-A를 건너뛰고 P-B(Seam 발견)로 직행한다. plan-extracts는 만들지 않는다.
