# Phase 8: 검증 (상세)

싼 검증부터 통과시킨다. 구조가 깨진 상태로 실행 테스트를 돌리는 것은 낭비다.

```
구조 → 정본 무결성 → 트리거 → 드라이런 → 실행 테스트 → Seam QA
```

> 스킬 테스트·트리거 검증 상세: `skill-testing.md` · 계약 대조 검증 상세: `seam-qa.md`

## 1. 구조 검증
- 에이전트 파일이 `.claude/agents/`에 올바르게, frontmatter(name, description, model:opus).
- 스킬 frontmatter(name, description), 본문 500줄 이내.
- 각 스킬에 **"컨텍스트 자산 참조" 섹션** 존재.
- **`.claude/commands/`에 아무것도 생성 안 됨.**
- 오케스트레이터 1개 존재, 실행 모드·데이터 흐름·에러 핸들링·테스트 시나리오·Phase 0 컨텍스트 확인 포함.
- **오케스트레이터 Phase 0에 "연기된 운영 결정 확인"이 배선됨** — 초기/부분/CE 재실행 판별과 나란히 있어야 한다. 메타 스킬 reference에만 있고 생성된 오케스트레이터 본문에 없으면 실패로 본다: 실제 개발 단계(오케스트레이터 직접 호출)에서 게이트가 돌지 않아, 조건이 충족되는 구현 시점에 연기 결정이 조용히 잊힌다.
- **제어 루프 결정이 PROGRESS에 기록됨** (상태 `연기`/`설치됨`/`생략`/`미설치` 중 하나 + 사유). 옵션이라도 "조용한 스킵"은 검증 실패로 본다 — 아무 언급이 없으면 아무도 검토하지 않았다는 뜻이다.
- **운영 결정에 상태 컬럼이 있고, 게이트가 상태를 전이시키는 규칙이 오케스트레이터에 있음** — 설치 후 `연기→설치됨` 갱신이 없으면 게이트가 매 실행 같은 결정을 다시 시드한다(무한 재시드).

## 2. 정본(CE) 무결성
- 각 seam 계약에 **seam-only 배너** 존재, 구현 상세 없음(design으로 이관 포인터).
- design 산출물이 계약을 **재기술하지 않고 링크**만.
- 표기 규약(`[정본]/[TODO]/[OPEN]/[ASSUMPTION]`) 일관.
- `docs/context/README.md`에 SoT·경계·변경 프로토콜·**오너십 맵**·변경 이력 존재.
- 같은 사실이 두 곳에 중복 존재하지 않음(drift 0).
- **오너십 정합:** 모든 seam에 소유자 정확히 1명, 소비자 1명 이상. 고아 정본·유령 seam 없음.
- 확정 계약이 `_workspace/`에만 남아 있지 않음(정본 승격 완료).

## 3. 트리거 검증
오케스트레이터·스킬 description에 대해 should-trigger(8~10) / should-NOT-trigger(near-miss 8~10). **near-miss의 절반 이상을 인접 seam으로 채운다** — CFHM에서 가장 흔한 오작동이 인접 seam 스킬의 오트리거다. 기존 스킬과의 충돌도 점검.

> 작성법: `skill-testing.md`

## 4. 드라이런
Phase 순서 논리성, 데이터 전달 경로 dead link 없음, 각 에이전트 입력이 이전 출력과 매칭, 에러 폴백 실행 가능, 계약 변경 통지 경로가 소비자 전원에게 닿는가.

## 5. 실행 테스트
스킬별 테스트 프롬프트 2~3개로 실제 실행. 가능하면 with-skill / without-skill 비교로 스킬의 부가가치를 확인한다 — 차이가 없으면 그 스킬은 삭제 대상이다.

> 방법론·평가·반복 개선: `skill-testing.md`

## 6. Seam QA (계약 대조)
정본을 assertion으로 전개하고 소유자·소비자 구현과 동시 대조한다. **측정(사실 수집)과 판정을 분리**하고, 불일치를 **V/G/S/D로 판정**한 뒤 규칙 등급에서 심각도(FAIL/WARN)를 정한다. "다름"에서 멈추지 않는다.

- 모든 계약 항목에 등급과 Rule ID가 있는가 (없으면 심각도를 못 매긴다)
- `SENSOR_ERROR`가 통과로 처리되지 않았는가 — **미측정을 정상으로 읽는 것이 가장 위험한 오류다**
- D(점진 이탈) 누적 카운트가 이어지고 있는가 (한 회차로는 판정 불가)

> 절차·판정 체계·경계면 버그 패턴·QA 에이전트 템플릿: `seam-qa.md`

## 7. 산출물 체크리스트
SKILL.md의 "산출물 체크리스트"를 그대로 점검한다. 특히 **P-B(Seam 발견)가 HE보다 먼저** 수행됐는지 확인.
